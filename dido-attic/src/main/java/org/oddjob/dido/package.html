Provide a framework for Data Input and Data Output.

The starting point for reading and writing data is a {@link Layout}.

Reading Data

When reading data the typical pattern is:

  1. Use {@link Layout#bind} to bind a {@link Binding} for creating Objects from the data.
  2. Use {@link Layout#readerFor} to create a {@link DataReader} from some {@link DataIn}.
  3. Call {@link DataReader#read} continually to read object from the data until null indicates there are no more objects to be read.

The sequence of events is: {@link DataIn} is read into {@link ValueNode} if the {@code Layout is such}. If the {@code Layout} is bound, this is passed to the binding.
